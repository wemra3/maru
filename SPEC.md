# maru — デザインフィードバック注釈ツール

## Goal
スクショ（クリップボード画像）に番号付きの注釈（円／角丸矩形）と修正テキストを付け、Claude Code に渡せる「該当箇所＋言語指示」を高速に作るデスクトップアプリ。Figma を開かず、線引き・採番の手間をゼロにする。

## Platform / Stack
- **Electron + Vite + React + TypeScript**（renderer に全 UI、main で OS クリップボード読み書き）
- パッケージは macOS .app（個人ツール、署名不要）
- 状態管理は軽量（Zustand か React state で十分。過剰な抽象化禁止）
- 描画オーバーレイは **SVG**（円・角丸矩形・リーダー線をベクターで）

## レイアウト / デザイン
- **完璧なグリッドレイアウト**。左 = キャンバスペイン、右 = インスペクタペイン
- キャンバス背景 = **少し濃いめのグレー**（#2b2b2e 前後）
- かっこいい・余白設計が綺麗・等幅グリッド。装飾過多禁止、機能美
- **コントロールは原則アイコンのみ（テキストラベル無し）**。ラベルを並べるのはダサいので避ける
- アイコンは**ちゃんとしたセット = `lucide-react`** を使う（自前の雑なアイコン禁止）
- アイコンのみでも分かるよう、**hover ツールチップ＋`aria-label`** を必ず付ける（視覚的なラベルではないので見た目はクリーンなまま、a11y は担保）

## 機能仕様

### キャンバス（左）
- **貼り付けボタン（アイコン）でクリップボード画像を貼る**（Electron `clipboard.readImage()`）。⌘V も併用可。
  - ※ 起動時の自動ペーストはしない（意図しない画像が入る不便を避ける）。明示的なペースト操作にする。
- ズーム（ホイール / ピンチ）・パン（ドラッグ）・移動可
- 画像が無い時はプレースホルダ（貼り付けアイコン＋「画像を貼る」）

### 注釈（円 / 角丸矩形）
- ツール ON（「赤丸」ボタン）状態で：
  - **クリック（ドラッグ小）= 円**（固定半径の点マーカー）
  - **ドラッグ（横長など範囲）= 角丸矩形**（始点〜終点、`rx` 角丸 2〜4px）
- 注釈生成と同時に **番号を自動採番**（①②…）、**右側に対応する入力欄を生成し、その欄へ即フォーカス**
- 既存注釈を再クリックで**削除**（採番は詰め直し）
- **最大 20 個**
- データモデル: `{ id, n, kind: 'circle'|'rect', x, y, w, h, text }`

### マーカー描画（Claude 認識性 + Hiroki の色覚を両立）
- **塗りつぶし禁止・細い輪郭リング（2〜3px）**。下の要素を隠さない
- **縁取り（白 or 黒の薄いハロー）**でデザインから浮かせる
- マーカー色は**設定可能**。デフォルトは赤緑色覚で見やすい高コントラスト色（マゼンタ/シアン系）を許容。名称は「赤丸」だが実装色は視認性優先
- **背景同化の自動回避（adaptive contrast）**：各マーカーの**下/周辺の画素をサンプル**し、マーカー色とのコントラストが閾値未満なら、**マーカー色＋縁取りを自動で対比色にシフト**する。背景画像と同色になっても勝手に見やすい色に変わる。Hiroki の赤緑色覚にも配慮（赤緑の対比に依存しない・輝度コントラストで判定）

### 番号バッジ配置（AI 認識性を最優先に再設計）
> 知見：AI(Claude) にとって最も確実なのは「高コントラストの図形＋**すぐ隣接した番号**」。L字リーダー線は人間向けで、AI にはむしろノイズ。座標番号は AI に効かない（中身で位置認識するため）。最強は冗長性＝視覚マーク＋番号＋テキストでの要素名。
- **デフォルト = 図形の角に小バッジ（隣接、最小オフセット）**。AI 認識に最適・実装も簡潔。コンテンツをほぼ隠さない位置（角の外側）に置く。
- **ガター＋L字リーダーは「バッジ同士が密集して衝突する時だけの自動フォールバック」**（人間の可読性向け、必須ではない・優先度低）。
- 右の番号別テキスト欄は「どうしたい」に加え**対象要素名を軽く書ける**運用を想定（出力に含めると AI 信頼性が上がる）。

### 番号ラベル自動配置 + L字リーダー線（唯一の非自明ロジック・独立モジュール）
- バッジは画像上でなく**左右のガター（余白）**に出す
- 対応図形の**中心 Y 順に縦に並べ**、重なりは**1次元の区間詰め**で押し下げる
- 図形が画像左寄り→左ガター / 右寄り→右ガター
- リーダー線 = **直交ポリライン（L字 / コの字、曲がり1〜2回）**でバッジ→図形の縁に接続
- **段階実装**：v1 簡易（図形角に小オフセットでバッジ、リーダー無し）→ 本実装（ガター＋L字）。本実装はレビュー厚め
- 2D 全方位の force-directed はやらない（過剰）

### キーボードショートカット
- **ツール切り替えをキーボードで**：注釈ツール / スポイト / パン（移動）を1キーで切り替え（例 `A`=注釈・`I`=スポイト・`V`=パン/選択、`Esc`=ツール解除）。キーは ergonomic に実装側判断可。
- 既存：⌘V=貼り付け、⌘N=新規ウィンドウ。コピー等もショートカット併設が望ましい。
- ※ 現在の v2 workflow 完了後の次パスで実装。

### インスペクタ（右）
- 番号別テキスト入力欄（注釈に 1:1、注釈生成で自動追加・削除で除去）
  - **入力欄は境界ドラッグでリサイズ可能**（広げたい時に高さ/サイズを変えられる。textarea の resize ハンドル等）。※次パスで実装
- **全体に対するテキスト欄**（特定図形に紐づかない全体コメント）
- **代表色パレット**：画像から代表色 10〜20 を抽出（median-cut / color-thief 等）してスウォッチ表示。クリックで HEX をコピー
- **スポイト**：画像上の任意ピクセルの色を取得（canvas getImageData）。**取得した色は代表色リストの末尾に追記して蓄積**していく（単一スロット上書きではなく、抽出色の後ろに並べる）。重複HEXは追加しない

### クリップボードコピー（3種）
1. **テキストのみ**：番号付きリスト＋全体テキスト
2. **注釈付き画像のみ**：画像＋リング＋番号を合成した PNG
3. **全体**：注釈付き画像＋テキスト
- テキスト出力フォーマット（Claude が食う形）:
  ```
  ① <text>
  ② <text>
  …
  全体: <globalText>
  ```

## v3 変更（UX再設計 / Figma風）

### A. コントロール再設計
- Cursor(選択/パン) / 注釈 / スポイト / ズームイン・アウト / カメラ(capture) を **キャンバス中央下のフローティングツールバー**に集約（Figma風）。
- **Export（コピー3種）は画面右上に単一配置**。現状の「上ツールバー＋右インスペクタ」の**二重配置を解消**（1箇所に統一）。
- インスペクタ見出しの無意味な「−」(Minus)アイコンを**削除**（ラベルは元々不要）。

### B. 注釈ツールの UX
- 注釈ツールのアイコンを **◯＋ポインターの合成アイコン**にする（lucideに無いのでカスタムSVG）。
- 注釈ツール選択中の **cursor を実際の◯スタンプ表示**に（十字でなく、これから押す円がマウス追従でプレビュー表示）。
- スタンプ（円/矩形）を置いた瞬間の **アニメーションを多少凝らせる**（pop/ripple 等。CSSキーフレームで、重い依存は足さない）。

### C. 入力 / ジェスチャ / ショートカット
- **トラックパッド2本指スクロール = パン**、**ctrl+wheel（ピンチ）= ズーム** に分離（macの標準挙動に合わせる）。
- **ピンチズームをさらにゆっくり**に（現状でもまだ速い）。
- **全てのボタンにキーボードショートカット**を割当、**ツールチップにショートカットキーを表示**（例「貼り付け ⌘V」）。
- **copy-all を再設計**：画像＋テキストの2形式併載（貼付で片方しか拾えず曖昧）をやめ、**注釈付き画像の下に番号テキストの凡例を焼き込んだ1枚の合成画像**を生成してコピー（自己完結・Claude/Slackにそのまま貼れる）。

### D. テキスト入力欄リサイズ
- 番号別テキスト入力欄を**境界ドラッグでリサイズ可能**（広げたい時に高さを変えられる）。

### E. 公開準備（後回し・別パス）
- **英語化**（i18n もしくは英語ベースに切替）。公開したいため。
- **electron-builder で .dmg 配布**（`dmg` ターゲット、署名なし可）。

### スコープ外（v1）
- ヒストリー（後日）
- 複数画像 / タブ
- クラウド保存

## 受け入れ基準（Acceptance Criteria）
- `npm run dev` で起動、`npm run build` が通る（TypeScript エラー 0）
- 起動時クリップボード画像が自動表示される
- クリック=円 / ドラッグ=矩形 が生成され、番号採番＋右欄フォーカスが動く
- 20個まで生成・削除・採番詰め直しが正しい
- 番号バッジが図形外に出て L字リーダーで繋がる（本実装フェーズ後）
- 代表色抽出・スポイト・コピー3種が動く
- ダークグレーのキャンバス＋綺麗なグリッドレイアウト

## ビルド段階（dev_loop / 自動 Workflow の単位）
1. **Scaffold**: Electron+Vite+React+TS、ダークな Grid シェル、`dev`/`build` 通る
2. **Canvas**: 起動時ペースト＋画像描画＋ズーム/パン
3. **Annotations**: クリック=円/ドラッグ=矩形、採番、右欄生成＋フォーカス、削除＆詰め直し（バッジは簡易=角オフセット）
4. **Labels**: 番号バッジのガター配置＋1次元詰め＋L字リーダー線（本実装）
5. **Exports**: 全体テキスト欄＋コピー3種（text / 注釈画像 / 全体）
6. **Colors**: 代表色抽出スウォッチ＋HEXコピー＋スポイト
7. **Polish**: グリッド/余白/色/マーカー縁取りの仕上げ、ビジュアル品質

各段階：実装 → `npm run build` で compile 検証 → コミット → レビュー（受け入れ基準照合）→ 必要なら修正。
※ GUI の見た目検証は最終的に Hiroki が行う（headless では compile/型/構造まで担保）。
